## Project Overview

**The Cottage Garden Companion** is a small, growing suite of **self-contained, single-file HTML tools** for the home gardener, dressed as a pressed-herbarium specimen sheet. There is **no build system, no dependencies, and no backend** — each tool is one `.html` file (markup + `<style>` + a vanilla-JS IIFE + its own embedded dataset) that opens directly from `file://`.

Current pages:

| File | Tool | What it does |
|---|---|---|
| `src/index.html` | The Cottage Garden Companion | Landing/hub page linking the collection. |
| `src/fertilizer.html` | The Fertilizer Plot | Turns any N–P–K ratio into a generative hand-drawn plant, then calculates feed dose, timing and cost. |
| `src/companion-planter.html` | The Companion Bed | 75+ plants with ranked companion + "keep apart" relationships; deep-links a recommended feed into the Fertilizer Plot. |

Tool sources live in `src/`. The published site lives in `docs/` (GitHub Pages serves it) and is generated by the `build` skill (`node .claude/skills/build/scripts/build.mjs`), which mirrors `src/` → `docs/` verbatim — there is still no compile step. Edit `src/`, never `docs/` (it is overwritten on each build).

`DESIGN.md` is the design-system source of truth (**do not edit**). `specs/` holds planning/phase docs and is git-ignored.

## How to Use This File

The sections below are **brief summaries**. Each links to a detail file under `.agents-docs/` holding the full content — **follow the link only for the area you're touching**, and read just what's relevant to the current task rather than everything up front.

Two rules of thumb: read **Conventions & Coupling Contracts** before changing anything shared across files (design tokens, the cross-tool nav, theme, deep-links, preset names), and read **Design System** before any visual change.

## Development & Verification

No build, lint, or automated test step — open the `.html` file in a browser, or serve the folder with any static server. Verification is a manual smoke-check in light **and** dark, at desktop **and** mobile widths, with no console errors.

Details: [Development & Verification](./.agents-docs/AGENTS-development-verification.md)

## Architecture

Every tool is a data-driven IIFE with four layers — **Data → Derive (pure) → Render → State/Wiring** — running one `render()` per state change. State persists in the URL hash + `localStorage`; graphics are inline SVG.

Details: [Architecture](./.agents-docs/AGENTS-architecture.md)

## Conventions & Coupling Contracts

House style is ES5-friendly vanilla JS (`"use strict"`, `var`, single IIFE per file). Several values are **copied verbatim across files** (design tokens, shared theme key, deep-link format, the six fertilizer preset names) and must never drift.

Details: [Conventions & Coupling Contracts](./.agents-docs/AGENTS-conventions-coupling-contracts.md)

## Design System

A warm paper-and-ink, flat/tonal pressed-herbarium aesthetic keyed to the N–P–K accent trio (leaf green · bloom pink · vigour gold), set in Fraunces (italic display) over Hanken Grotesk (UI). Full light/dark token set. `DESIGN.md` is canonical.

Details: [Design System](./.agents-docs/AGENTS-design-system.md)
